# MVP Documentation

This folder contains the mandatory documentation required for the **Minimum Viable Product (MVP)** phase of the project. 

## Purpose of the MVP
The MVP phase is designed for rapid prototyping to demonstrate capabilities. It usually involves building a demo page with minimal or no backend services (often relying on mocked data or simple endpoints to show that we are capable of delivering the solution). 

> [!NOTE]
> Unless otherwise specified by the client, the MVP should utilize the **Orionex Design Theme** (see `docs/web-brand-guide.md`).

## Mandatory Documents

The following documents must be completed for the MVP phase:

1. **`project-overview.md`**: Defines the scope, goals, and limitations of the MVP.
2. **`user-journey-map.md`**: High-level mapping of the user's experience for the demo.
3. **`user-flow-diagram.md`**: Specific step-by-step screen flows for the MVP features.
4. **`use-case-diagram.md`**: Identifies system actors and their primary interactions covering the demo scope.
5. **`api-contract.md`**: The API contracts utilized by the frontend. In the MVP phase, these are often mocked or vastly simplified.
6. **`database-schema.md`**: The database structure. If the MVP operates without a database, document the mocked data structures here.

## Project Focuses (Optional Documents)
Depending on the specific **project focuses**, other non-mandatory documents may be required even during the MVP phase (e.g., if a client specifically wants to see a proof of concept for a compliance feature). 

If such documents are needed, they should be created in this directory, but they are not included as default templates.
